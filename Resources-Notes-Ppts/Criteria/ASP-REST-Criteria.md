# ASP REST QC Criteria

## REST

| **Priority** | **Criteria**                                                             |
| --------- | ------------------------------------------------------------------------------------------------------------ |
| Required  | Describe the purpose of REST and RESTful design.                                                             |
| Required  | Describe the purpose of HTTP messaging.                                                                      |
| Required  | Describe the HTTP request/response lifecycle.                                                                |
| Required  | Describe HTTP request methods (verbs).                                                                       |
| Required  | Describe HTTP response code classes (1xx,2xx,3xx,etc.)                                                       |
| Required  | Describe the REST principles.                                                                                |
| Required  | Be capable of sending a GET request to an open source REST API using curl or Postman                         |
| Preferred | Describe URL conventions when using REST.                                                                    |
| Preferred | Describe SOA (Service Oriented Architecture) and be capable of diagramming the components of a sample system |
| Preferred | Describe the difference between authorization and authentication.                                            |
| Preferred | Be capable of sending a POST request to a REST API using curl or Postman and populating the request body     |
| Preferred | Build a RESTful web service using a popular framework (e.g. Spring, Flask, Express)                          |

## ASP.NET Core

| **Priority** | **Criteria**                                                             |
| ------------ | ------------------------------------------------------------------------ |
| Required     | Describe the HTTP pipeline                                               |
| Required     | Implement controllers and action methods.                                |
| Required     | Describe the purpose and types of HTTP response codes.                   |
| Required     | Implement an API service.                                                |
| Required     | Demonstrate functional kowledge of Data Transfer Objects, and their use. |
| Required     | Describe and implement a Minimal API endpoint.                           |
| Required     | Describe the function of HTTP method annotations.                        |
| Preferred    | Implement model binding effectively.                                     |
| Preferred    | Describe and implement data validation using annotations.                |
| Preferred    | Demonstrate the use of automatic mapping for objects and DTOs.           |
| Preferred    | Implement native ASP.NET middleware, such as Logging or Identity.        |
| Preferred    | Implement HTTP response codes effectively.                               |
| Stretch      | Implements third party or custom filters and middleware.                 |
| Stretch      | Demonstrate understanding of caching in an API.                          |
| Stretch      | Implement an API which consumes a 3rd party API.                         |
