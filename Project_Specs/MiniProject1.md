# Mini-Project-1 (Solo)

For this project we will create an ASP.NET API, implementing the CQRS pattern with MediatR. 

## Specs

- ASP.NET API w/EF Core
- CQRS pattern
- Uses MediatR
- Full CRUD Operations
- SQLServer Database(s) - Atleast one M-M relationship
- Implements Identity
  - Atleast two user roles, one "regular" role and one "admin" role. 
- API and database containerized and running locally.

## Considerations

The project's theme can be anything as long as it's sufficiently complex as to satisfy the database schema/auth scheme
requirement. Eventually you will design a React front end that will use this API. For this iteration, we will focus on
the backend. 

## Sample Ideas:
- Concert/event platform (Dice, Eventbrite, etc.)
- Fitness coaching/workout tracker application (Hevy or Strong as examples.)
- Freelancer/project bidding system (Fiver or Upwork as examples)
- IT Ticket Management system (Jira or similar systems)
- Custom PC Build Configurator (think PCPartPicker)
